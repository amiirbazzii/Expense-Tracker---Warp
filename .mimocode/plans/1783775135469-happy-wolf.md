# Fix: Loan Payment - Stale Data & Duplicate Expenses

## Root Cause

`PayInstallmentSheet.handleSubmit` creates the same expense **twice** via two independent paths:

1. `localDataStore.addExpense()` — saves to local IndexedDB + **enqueues** a Convex mutation to the `MutationQueueManager`
2. `createExpenseMutation()` — calls Convex directly (when online)

When the `SyncEngine` drains the queue, it calls `createExpense` on Convex **again** (path 1's enqueued mutation). Result: **2 expenses on Convex**. Then `HydrationService` fetches from Convex, can't match them to the local expense (local uses `local_xxx` ID, Convex uses its own `_id`), and inserts both as new local entities. Net result: **2+ duplicate expenses everywhere**.

The "stale data" symptom is caused by this duplication — the user sees one expense from the direct mutation (correct data) and another from the hydration/pending queue (potentially stale or from a different sync cycle).

## Fix

### File: `src/features/loans/components/PayInstallmentSheet.tsx`

**Remove the direct Convex mutation calls** in `handleSubmit`. The local-first store already handles enqueuing:

1. `localDataStore.addExpense()` — creates expense locally + enqueues `expenses:createExpense`
2. `localDataStore.payInstallment()` — increments paid count locally + enqueues `loans:payInstallment`

No direct `createExpenseMutation()` or `payInstallmentMutation()` calls needed — the `SyncEngine` queue handles sync when online.

**Changes to `handleSubmit`:**
- Remove the entire `if (navigator.onLine) { ... }` block (lines 182-198)
- Keep only: `localDataStore.addExpense(...)` + `localDataStore.payInstallment(...)`

**Cleanup unused imports/variables:**
- Remove `useMutation` from `convex/react` import
- Remove `api` import from convex generated
- Remove `createExpenseMutation`, `payInstallmentMutation` variables
- Keep `useQuery` (still needed for cards/categories/forValues queries)
- Keep `useAuth` (still needed for `token` in queries)

### Verification
1. Open loan page, click Pay Installment on a loan
2. Change the date to yesterday, select a different card
3. Click Pay — verify expense appears with the **changed** date and card
4. Check expenses list — verify only **one** expense appears (no duplicates)
5. After sync completes — still only one expense

# Fix: Card Creation Duplicate

1. `localStorageManager.saveCard()` — saves to local IndexedDB
2. `localDataStore.addCard()` — saves to local IndexedDB + enqueues Convex mutation
3. `addCardMutation()` — calls Convex directly → **duplicate after sync**

**Cleanup:**
- Remove `addCardMutation` variable (line 46)
- Remove unused `useMutation` import if no other mutations remain
