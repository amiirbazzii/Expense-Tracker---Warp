# Fix: Stale `.next` build cache

## Problem
`Cannot find module './216.js'` — the `.next/server/` directory has stale webpack chunks that no longer match the current source. This is a common Next.js dev server issue after code changes or interrupted builds.

## Plan
1. Delete the `.next` folder: `rm -rf .next`
2. Restart the dev server: `npm run dev`

## Verification
- Dev server starts without `MODULE_NOT_FOUND` errors
- App loads in browser without 500 errors
